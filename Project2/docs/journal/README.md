## 如何写日志

每次重大决策、重构或踩坑后，创建新的日志文件：

```markdown
# YYYY-MM-DD - 简短描述

## 背景
为什么需要做这个改动。

## 决策
做了什么决定，为什么。

## 替代方案
考虑过的其他方案，以及为什么没有选择。

## 结果
改动后的效果。

## 教训
下次可以做得更好的地方。

## 相关链接
```
